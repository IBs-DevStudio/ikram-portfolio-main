# Why I Started Writing: Building Beyond Code

At some point, I realized building projects wasn’t enough.

I was solving problems, experimenting with AI, creating tools — but none of that thinking was visible. This blog is where I document that journey.

Not just what I build, but *why I build it*.

---

## The Shift

Most people build projects to showcase skills.

I build to rethink how things should work.

Whether it’s:
- Turning static learning into interactive AI companions (Cogniva)
- Making screen recording faster and frictionless (RecordKaro)
- Helping developers understand their codebases instantly (GitIntellect)

Every project starts with one question:

> “Why is this still done the old way?”

---

## Why This Blog Exists

This isn’t just a blog.

It’s a place where I:
- Break down problems before solving them  
- Share how AI can improve everyday workflows  
- Document decisions behind my projects  
- Explore better, faster, more intuitive ways to build  

---

## What You’ll Find Here

- **Real Problem Solving**  
  Not tutorials for the sake of tutorials — but solutions that come from actual challenges.

- **AI + Product Thinking**  
  How AI is changing the way we design, build, and interact with software.

- **Behind the Projects**  
  The thinking, trade-offs, and iterations behind tools I create.

---

## The Goal

To move from:
> just building features  

to:
> building meaningful experiences  

And from:
> writing code  

to:
> solving problems with clarity

This is just the beginning.
